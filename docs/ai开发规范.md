# AI 开发规范文档（适用于 AI 智能客服系统）

## 一、文档目的
为保证 AI 模块（包括 Dify 工作流、Gemini 模型接入、知识库问答、意图识别等）在研发、联调、上线中的一致性、可维护性与安全性，特制定本规范。本规范适用于：
- 后端 AI 调度层（AI 服务网关）；
- 外部模型调用层（Gemini、Dify 等）；
- Prompt 设计与版本管理；
- AI 回答质量评估与监控。

---

## 二、AI 模块职责

| 模块 | 主要功能 | 接口方向 |
|------|------------|------------|
| AI 网关（AI Gateway） | 对接多个模型（Gemini、Dify、OpenAI），统一鉴权、限流、日志。 | 内部调用 |
| 工作流引擎（Dify Workflow） | 执行提示词链、知识检索、意图判断、置信度输出。 | 内外部调用 |
| 知识库（Knowledge Base） | FAQ 文档、游戏规则、客服政策等内容管理。 | 读写 |
| 意图识别模块（Intent Classifier） | 将用户输入分类为：咨询 / 投诉 / 充值 / 封号 / 建议 等。 | 内部调用 |
| 置信度计算模块（Confidence Evaluator） | 根据模型输出评分判断是否转人工。 | 内部调用 |
| 日志与审计模块 | 保存每次 AI 调用的上下文、提示词、置信度、输出结果。 | 异步写入 |

---

## 三、AI 调用统一规范

所有模型调用（无论 Gemini、Dify、或第三方）必须经过统一的 AI Gateway API。

### 接口示例
```json
POST /api/ai/generate
{
  "session_id": "abc123",
  "user_input": "为什么我被封号了？",
  "context": {
    "history": [],
    "player_info": {}
  },
  "model": "gemini-1.5-pro",
  "temperature": 0.3,
  "top_p": 0.8,
  "max_tokens": 800
}
```

返回格式：
```json
{
  "answer": "您好，您的账号因违规行为被系统暂时封禁3天。",
  "intent": "封号咨询",
  "confidence": 0.82,
  "tokens_used": 240,
  "model": "gemini-1.5-pro",
  "from_cache": false
}
```

## 四、提示词（Prompt）设计规范

### 结构模板
```markdown
# Role
你是第七大道游戏的智能客服助手，语气友好、表达简洁。

# Task
根据用户的问题，从知识库内容中提取答案，如无匹配请保守回复。

# Rules
1. 不编造不存在的政策；
2. 如回答不确定，请说明需要人工客服；
3. 保持每条回复 60 字以内；
4. 输出简体中文；
5. 不泄露系统内部信息。

# Format
{
  "answer": "...",
  "intent": "...",
  "confidence": 0.x
}
```

### 编写原则
1. 明确角色（Role）；
2. 明确任务（Task）；
3. 限定信息来源（Rules）；
4. 约束输出格式；
5. 版本控制（每次更新 Prompt 必须递增版本号并附变更说明）。

## 五、置信度与转人工规则

| 置信度区间 | 系统行为 |
|-------------|-----------|
| ≥ 0.85 | 直接回复玩家 |
| 0.6–0.85 | 回复同时提示“如不满意可转人工” |
| < 0.6 | 自动转人工排队 |
| null | AI异常 → 立即转人工 |

## 六、安全与合规
1. **隐私保护**：禁止在 Prompt 或训练语料中出现真实玩家账号、手机号、订单号。  
2. **内容过滤**：AI 回复需经过关键词黑名单与敏感词过滤。  
3. **数据脱敏**：所有日志中用户ID需hash化（如 SHA256(uid)）。  
4. **外部调用审计**：AI Gateway 记录模型调用时间、Token 消耗、错误率。  
5. **Prompt 审核制度**：每次修改需走人工审核流程（避免 Prompt 注入风险）。

## 七、AI 编程行为约束规范

### 13.3 基本原则

| 序号 | 原则 | 说明 |
|------|------|------|
| 1️⃣ 精准修改原则 | AI 仅能修改用户明确指示的代码段或文件范围，不得自行重构无关模块。 |
| 2️⃣ 代码不可外传原则 | 禁止 AI 工具将项目源代码、API 密钥、配置文件上传到任何外部云端或模型接口。 |
| 3️⃣ 本地执行优先 | 所有 AI 代码生成与分析应优先在本地模型/离线模式或公司授权账户下进行。 |
| 4️⃣ 不得擅自提交代码 | AI 工具不得自动执行 `git commit` / `git push` 等操作。 |
| 5️⃣ 版本差异清晰 | 使用 AI 修改代码后，必须通过 `git diff` 检查修改范围与逻辑正确性。 |
| 6️⃣ 不得自动生成机密配置 | 禁止 AI 自动创建、填写 `.env`、`API_KEY`、数据库密码等敏感文件。 |
| 7️⃣ 人工审核必经 | AI 输出的任何逻辑改动（接口、算法、数据库迁移）必须经人工审核通过后方可合入主分支。 |
| 8️⃣ 依赖包需验证来源 | AI 建议安装的新依赖必须人工验证版本与安全性。 |
| 9️⃣ 规范命名与注释 | AI 生成代码必须符合命名规则并附中文注释说明修改意图。 |
| 🔟 数据脱敏原则 | AI 生成的日志或测试数据不得包含真实用户数据。 |

### 禁止性行为
- 在公开对话（ChatGPT网页版等）中粘贴项目代码或密钥。  
- 让 AI 自动“优化整个系统”或全局重构项目。  
- 使用 AI 工具自动执行 `git push` / `npm install` 命令。  
- 修改配置文件（`package.json`、`.env`、`vite.config.js` 等）未经审核。  

### 正确的 AI 使用流程
1. 明确任务范围 → 指定修改文件与功能点。  
2. 生成代码草案 → 审查 AI 输出的逻辑与注释。  
3. 手动测试 → 确认编译通过与功能正确。  
4. 手动提交代码 → 禁止自动提交。  
5. 代码审查 → Pull Request 审核后合并。  

### AI Prompt 编写约束
- 必须明确文件与修改范围；
- 禁止模糊指令（如“帮我优化整个项目”）；
- 推荐模板：
  ```
  请仅修改 src/components/ChatPanel.jsx，
  在输入框下方新增快捷回复按钮，
  不要改动其他文件、不删除任何已有代码，
  并添加中文注释说明修改内容。
  ```

### 责任与审计
- 所有 AI 协助产生的代码最终责任由提交人承担；
- AI 工具调用与修改记录纳入审计范围；
- 发现违规行为将暂停 AI 权限并进行追溯。
